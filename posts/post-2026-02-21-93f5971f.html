제2강. 개발 환경 구축: 5분 만에 기지 세우기

OpenClaw를 사용하여 자신만의 무인 자동화 기지를 구축하는 첫 단계는 개발 환경을 올바르게 설정하는 것입니다. 본 강의에서는 설치부터 게이트웨이 실행까지의 모든 기술적 단계를 설명합니다.

1. 사전 요구 사항
OpenClaw는 Node.js 환경에서 작동합니다. 최신 안정화 버전(LTS)이 설치되어 있는지 확인하십시오.


node -v




2. CLI 도구 설치
터미널에서 npm을 통해 OpenClaw 명령줄 도구를 전역으로 설치합니다.


npm install -g openclaw@latest




3. 시스템 초기화 및 워크스페이스 생성
설치가 완료되면 시스템을 초기화해야 합니다. 이 과정에서 설정 파일(~/.openclaw/openclaw.json)과 에이전트가 작업할 전용 공간인 워크스페이스가 생성됩니다.


openclaw setup




대화형 마법사를 통해 설정을 진행하고 싶다면 다음 명령어를 사용하십시오.


openclaw setup --wizard




4. 게이트웨이 실행
모든 설정이 완료되었다면 에이전트의 심장인 게이트웨이를 구동합니다. 게이트웨이는 웹소켓(WebSocket) 서버 역할을 하며 에이전트와 사용자, 그리고 시스템 노드 간의 통신을 관리합니다.


openclaw gateway start






openclaw status




5. 대시보드 접속
브라우저를 통해 현재 에이전트의 상태를 시각적으로 확인하고 관리할 수 있는 대시보드에 접속합니다.


openclaw dashboard




이제 여러분의 로컬 시스템에 강력한 에이전트 환경이 구축되었습니다. 다음 강의에서는 에이전트에게 실제 실행 능력을 부여하는 Skills와 Tools 시스템에 대해 알아보겠습니다.
